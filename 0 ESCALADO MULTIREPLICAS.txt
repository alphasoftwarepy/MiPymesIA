python# auth.py - Solo cambia la conexión
import os
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker

# Detecta automáticamente el tipo de DB
DATABASE_URL = os.getenv(
    "DATABASE_URL", 
    "sqlite:///./users.db"  # Fallback para desarrollo
)

# Si es PostgreSQL desde Easypanel
if DATABASE_URL.startswith("postgres://"):
    # Easypanel usa postgres://, pero SQLAlchemy necesita postgresql://
    DATABASE_URL = DATABASE_URL.replace("postgres://", "postgresql://", 1)

engine = create_engine(DATABASE_URL)
SessionLocal = sessionmaker(bind=engine)
Migración de SQLite a PostgreSQL:
python# migration_script.py - Ejecuta una vez
import sqlite3
import psycopg2
import os

# Lee de SQLite
conn_sqlite = sqlite3.connect('users.db')
cursor = conn_sqlite.cursor()
cursor.execute("SELECT * FROM users")
users = cursor.fetchall()

# Escribe en PostgreSQL
conn_pg = psycopg2.connect(os.getenv("DATABASE_URL"))
cursor_pg = conn_pg.cursor()

for user in users:
    cursor_pg.execute(
        "INSERT INTO users (username, password_hash, created_at) VALUES (%s, %s, %s)",
        user
    )

conn_pg.commit()

## Alternativa: SQLite con solo 1 réplica siempre

Si prefieres **NO cambiar de base de datos** (mantener SQLite):

**Limitaciones:**
- ✅ Funciona perfecto con 1 réplica
- ❌ NO puedes escalar a múltiples réplicas
- ✅ Simplicidad total (tu código actual)

- Aumenta los recursos del VPS (más RAM/CPU)
